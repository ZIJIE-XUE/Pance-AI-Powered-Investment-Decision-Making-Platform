## 为什么做这个项目？

在本科期间，我注意到两个现象的交汇：

**第一个现象**：中国有超过 2 亿基民，但绝大多数个人投资者的决策方式仍然停留在"看新闻买基金、听朋友推荐赎回"。行为金融学告诉我们，散户在恐慌时不敢买（损失厌恶）、在狂热时追涨（羊群效应）、在正常波动中过度反应（锚定效应）——这些认知偏差系统性地侵蚀了投资收益。

**第二个现象**：现有的智能投顾（Robo Advisor）产品，无论是国内的蚂蚁财富"帮你投"还是 Betterment / Wealthfront，本质上做的都是同一件事——问几个问题，给你一个静态的股债比例，然后就不管了。它们缺少一个关键环节：**根据市场状态动态调整投资决策**。

于是我问了一个问题：**能否将市场估值信号引入定投策略，建立一个基于规则的、数据驱动的、有行为金融学理论支撑的量化投资决策系统？**

这就是磐策（PánCè）的起点——"稳如磐石，策定乾坤"。

## 我做了什么

磐策是一个完整的 AI 驱动智能投顾平台，核心创新是**温度定投系统**——一个围绕市场估值温度信号构建的量化 DCA 引擎。整个系统包含：

### 旗舰模块：温度定投系统

这是项目的核心和最大亮点，围绕一个简单但有实证基础的假说构建：**市场估值温度对未来收益具有预测能力**——温度越低，未来期望收益越高。

围绕这个假说，我构建了**五层分析框架**：

1. **历史回测**：温度定投 vs 普通定投 vs 一次性买入，在同一时间段内跑完整模拟。温度定投通过现金池机制自动调节每期投入金额——低估时从现金池提取加仓，高估时少投积存现金。

2. **信号验证**：在回测之前，先用 Pearson 相关分析验证温度信号是否真的具有预测能力。将月度温度与未来12个月收益做散点图回归，按温度分桶统计各区间平均收益。这为整个策略提供了实证基础——不是"我觉得有效"，而是"数据显示有效"。

3. **Walk-Forward 样本外检验**：这是量化策略验证的黄金标准。将数据拆分为训练窗和测试窗，在训练窗内通过网格搜索（PE权重 × 策略档位）最大化夏普比率，然后将锁定参数在从未接触过的测试窗上只看一遍。如果样本外仍然跑赢，说明策略不是过拟合。

4. **市场环境分解**：将回测拆分为牛市、熊市、震荡市三个子区间，分别展示温度定投在各环境下的表现。诚实地呈现 trade-off——牛市中一次性买入最优、熊市中温度定投领先——让使用者理解策略的边界。

5. **行为金融学框架**：将系统定位于"规则化的行为金融干预工具"——温度定投通过规则化操作帮助投资者克服损失厌恶（低估自动加仓）、羊群效应（高估强制少投）和锚定效应（适中维持基准）。

### 完整 Robo Advisor 工作流

除旗舰模块外，磐策还包含完整的智能投顾管线：

- **风险测评**：12题科学问卷 → 5级风险等级（保守→激进）
- **组合优化**：两级优化——风险等级决定大类资产权重 → PyPortfolioOpt 在30+只ETF中均值-方差优化
- **Monte Carlo 模拟**：10,000条GBM路径模拟5/10/20年收益分布，含VaR/CVaR尾部风险
- **AI 分析引擎**：本地规则引擎，即时生成配置理由、风险识别、情景分析
- **PDF 报告生成**：7章专业报告，ReportLab + Plotly，中文完美渲染
- **市场仪表盘**：实时A股指数行情、行业热力图、PE温度计、ETF对比

### 技术决策

- **100% 免费数据源**：AKShare（新浪 + 中证指数 + 东方财富）+ Yahoo Finance，无需任何 API Key
- **AI 完全本地运行**：规则引擎 + 本地计算，不依赖外部 LLM——保证隐私和可用性
- **Streamlit 全栈**：61个Python文件，13,000+行代码，分层架构（engine / services / agents / ui / db）
- **数据可靠性处理**：多数据源主备切换（Sina → EastMoney）、指数符号双格式适配（sh/sz前缀）、缓存策略分层

## 这个项目展示了什么

### 对 BA/FinTech 项目而言

- **量化方法论素养**：不只是"写了个回测"，而是构建了信号验证 → 样本外检验 → 环境分解 → 行为金融的完整分析链条。每一步都有方法论依据。
- **实证思维**：不满足于"回测跑赢"，而是追问"温度信号真的有效吗？""样本外还能赢吗？""什么情况下会输？"——这种追问正是研究生阶段需要的。
- **Trade-off 分析能力**：诚实展示策略的边界——牛市中会落后、样本外会衰减——不做 cherry-pick。商学院投资学第一课就是"没有免费的午餐"。
- **行为金融学的应用**：将理论（损失厌恶、羊群效应、锚定效应）转化为实际的量化规则，展示了理论与实践的结合能力。

### 对技术面试而言

- **全栈工程能力**：从数据获取（AKShare多源适配）到计算引擎（回测/优化/Monte Carlo）到前端（Streamlit + Plotly）到报告生成（ReportLab PDF），独立完成完整产品。
- **Python 生态的深度使用**：NumPy/SciPy（统计检验）、Pandas（数据处理）、Plotly（可视化）、SQLAlchemy（数据库）、Pydantic（配置管理）
- **工程质量意识**：分层架构、缓存策略、错误处理、多数据源容灾
