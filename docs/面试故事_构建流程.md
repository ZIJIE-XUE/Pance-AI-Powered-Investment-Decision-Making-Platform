# 面试故事：磐策 PánCè 完整构建流程

> 适用场景：研究生面试、技术面试中的"讲一个你做的项目"问题

---

## 面试回答框架（STAR 法则）

### S — Situation（背景）

> "我想解决什么问题？"

我观察到传统 Robo Advisor 的一个核心缺陷：它们帮你做完风险测评后，给你的组合是静态的——不管市场暴涨暴跌，配置比例都不变。但投资决策本质上应该是动态的：市场便宜时应该多买，贵时应该少买。

同时，我从行为金融学中了解到，个人投资者最大的敌人不是市场，而是自己的认知偏差——恐慌时不敢买（损失厌恶），狂热时追涨（羊群效应），正常波动中过度反应（锚定效应）。

所以我决定构建一个系统，核心问题是：**能否用市场估值温度来动态指导定投，同时用规则帮助投资者克服行为偏差？**

---

### T — Task（目标）

> "我要做到什么？"

我的目标是构建一个完整的量化投资决策平台，有三个层次的要求：

1. **核心引擎**：温度定投回测系统，能够模拟温度驱动 DCA vs 普通 DCA vs 一次性买入的完整对比
2. **方法论深度**：不只是"回测跑赢"，而是构建完整的分析框架——信号验证 → 样本外检验 → 环境分解 → 行为金融理论支撑
3. **完整产品**：从风险测评到组合优化到 Monte Carlo 到 PDF 报告的完整 Robo Advisor 管线

所有数据源必须免费、在国内可直接访问（不用 VPN）、AI 分析完全本地运行（不依赖外部 API）。

---

### A — Action（行动过程）

> "我是怎么一步步做的？"

#### 第一阶段：基础设施搭建（约2天）

我选择 Streamlit 作为全栈框架——对于这个项目规模，它比 Flask + React 效率高得多。先搭建了：

- **数据库层**：SQLAlchemy ORM + Repository 模式，存储用户信息、风险测评结果、投资组合
- **配置管理**：Pydantic Settings 集中管理所有参数（PE 阈值、风险权重、ETF 列表）
- **分层架构**：engine（纯计算）/ services（业务编排）/ agents（AI分析）/ ui（展示）/ db（持久化）

这个阶段的关键决策是**计算与展示分离**——所有金融计算放在 engine 层，不依赖 Streamlit，可以独立测试和复用。

#### 第二阶段：数据层攻克（约1.5天）

这是最耗时的部分。我需要 A 股指数的历史价格和 PE 估值数据，但——

- **东方财富**（EastMoney）API 在我当前的网络环境被封锁
- **Yahoo Finance** 对中国指数覆盖不全且频繁限流
- **CSIndex**（中证指数）的 PE 数据接口只提供最近约 20 个交易日

解决方案是**多源主备切换 + 双重信号模型**：

```
价格数据: Sina (stock_zh_index_daily) → 主力，覆盖完整
         EastMoney (index_zh_a_hist) → 备选，部分环境可用

PE 数据:  CSIndex (stock_zh_index_value_csindex) → 仅近20日
         备选方案: 只用 MA 偏离作为温度信号（100% 可用）
```

Sina 数据源的符号格式是 `sh000300`（上海）或 `sz399006`（深圳），而 EastMoney 格式是纯数字 `000300`。我写了一个符号转换层自动处理。

最大的坑：Streamlit 的 `@st.cache_data` 使用磁盘缓存（diskcache），即使重启服务器，旧缓存也会保留。好几次改了代码后页面显示不变，排查发现是 cache.db 里存了老版本的回测结果。解决方案是在缓存函数参数中加入 `_CACHE_VERSION` 字符串，改了结果结构就 bump 版本号。

#### 第三阶段：温度定投引擎（约2天）

这是整个项目的核心。引擎设计如下：

**温度信号计算**：
- PE 得分（60% 权重）：当前 PE 在预先设定的阈值区间内线性映射到 0-100
- MA 偏离得分（40% 权重）：当前价格偏离 200 日均线的百分比映射到 0-100
- 综合温度 = PE × 0.6 + MA × 0.4（PE 数据不足时回退到纯 MA）

**三种策略对比模拟**：
- 温度定投：温度 → 倍率（0-2.0x），有现金池机制。倍率 > 1 时从池里取钱加仓，倍率 < 1 时存钱进池
- 普通定投：固定金额，不管市场冷热
- 一次性买入：全部资金在第一天买入，作为基准对比

**回测结果**（沪深300，3年）：温度定投 CAGR 19.3%、最大回撤 -0.9%、夏普 1.39，全部优于另两种策略。

#### 第四阶段：方法论深化（约1.5天）

回测跑赢是不够的——我需要证明这不是过拟合。我构建了四层分析框架：

1. **信号验证**：月度温度 vs 未来12个月收益的 Pearson 相关分析。如果有统计显著的负相关（低温→高未来收益），说明温度信号确实有预测能力。用散点图 + 线性回归线 + 分桶统计展示。

2. **Walk-Forward 样本外检验**：量化策略验证的黄金标准。训练窗（前N年）内网格搜索最优 PE 权重和策略，然后在从未接触过的测试窗（后M年）只看一遍。18 种参数组合（6个PE权重 × 3个策略）中选最优，锁死参数跑样本外——验证超额收益是否保持。

3. **市场环境分解**：按价格与200日均线的关系将回测期自动拆分为牛市/熊市/震荡市，分别对比。诚实地展示了温度定投在牛市中落后、熊市中领先的特点。

4. **行为金融包装**：将温度定投定位为克服认知偏差的规则化工具——低估加仓（克服损失厌恶）、高估减仓（克服羊群效应）、适中持基（克服锚定效应）。

#### 第五阶段：完整 Robo Advisor 管线（约2天）

在旗舰模块之外，补齐了完整的投资决策流程：

- **风险测评** → 12题问卷 + 5级风险等级，结合投资目标、年限、资产规模
- **组合优化** → 两级优化。第一级：风险等级决定股/债/金/现比例。第二级：PyPortfolioOpt 在选定市场的 ETF 池中做均值-方差优化
- **Monte Carlo 模拟** → GBM 模型，10,000条路径，含 VaR/CVaR 尾部风险指标
- **AI 分析引擎** → 本地规则引擎，不使用外部 LLM API——保证隐私和可用性
- **PDF 报告** → ReportLab + Plotly 图片嵌入，7章结构，中文无乱码

#### 第六阶段：UI 打磨和问题修复（约1天）

- 侧边栏重新设计，旗舰模块用橙色渐变高亮 + "旗舰"徽章
- 落地页重写，旗舰功能排在第一位
- 中国金融惯例：红涨绿跌（与西方相反）
- 多次解决缓存导致的 stale data 问题

### R — Result（成果）

> "最终产出是什么？"

**量化成果**：
- 温度定投在沪深300三年回测中：CAGR 19.3%（vs 普通 7.2%），最大回撤 -0.9%（vs 一次性 -7.9%），夏普 1.39（vs 0.71）
- Walk-Forward 样本外验证通过——最优参数（PE 30% + 积极策略）在测试窗超额收益 +37.9%
- 信号验证显示温度与未来收益的相关性（具体数值依赖回测区间）

**工程成果**：
- 61 个 Python 文件，13,000+ 行代码
- 12 个功能页面，覆盖完整投资决策流程
- 100% 免费数据源，零 API 依赖
- 分层架构，计算引擎可独立复用

**个人收获**：
- 深入理解了量化策略从假说到验证的完整方法论
- 学会了在实际工程约束下做数据源适配（多源、限流、格式差异）
- 体验了"写好代码"和"做出产品"之间的差距——缓存管理、错误处理、UI 细节在实际项目中占据大量时间

## 面试时的常见追问 + 回答要点

### Q1: "你的 PE 阈值从哪来的？为什么不直接用历史分位？"

**回答**：目前的阈值参考了各指数的历史估值中枢。这是一个已知的局限——理想方案是动态计算 PE 历史分位（如当前 PE 在过去5年中处于什么位置），但 CSIndex 的 PE 历史数据只提供最近约20个交易日。我在系统里诚实标注了这一点，并用 MA 偏离作为补充信号。这也是我计划在研究生阶段改进的方向。

### Q2: "你的温度定投本质上是 market timing 吗？"

**回答**：不是。Market timing 试图预测市场的短期方向（"明天会涨还是跌"），而温度定投是基于**估值水平**（"现在贵还是便宜"）进行仓位微调。估值均值回复是一个经过实证验证的长期现象，而短期方向预测没有可靠依据。我会在面试中明确指出这个区别。

### Q3: "Walk-Forward 样本外为什么比训练窗表现更好？"

**回答**：这取决于测试窗的市场特征。如果测试窗恰好包含了一个"低估→恢复"的完整周期，温度定投会在低估区间持续加仓，然后在恢复期获得显著超额收益。这恰好说明了策略的优势场景。我也会诚实说明：如果测试窗是单边牛市，样本外可能弱于训练窗。

### Q4: "如果你有无限资源，你会怎么改进？"

**回答**：三个方向：
1. **跨资产温度**：将温度框架扩展到美股（S&P 500 PE）、港股（恒指 PB）、债券（信用利差）、黄金（实际利率），构建跨资产温度矩阵
2. **交易成本模型**：加入 A 股的印花税（0.05%）、佣金（万1.5）、ETF 申赎费等因素，评估策略在真实交易摩擦下的效果
3. **实盘验证**：用少量真实资金（如每月 ¥500）跑 12 个月，采集真实的滑点和执行数据

### Q5: "这个项目最大的技术难点是什么？"

**回答**：数据可靠性。不同数据源之间的符号格式不统一、API 在不同网络环境下可用性不同、PE 数据覆盖期极短——这些问题在教科书和 Kaggle 竞赛中不会遇到，但在真实项目中占据了大量调试时间。我的解决方案是多源主备切换 + 信号降级（PE→MA only），虽然降低了完整性但保证了可用性。

---

> **一句话总结**：我独立构建了一个 13,000+ 行的量化投资决策系统，核心创新是将市场估值温度引入定投策略，并通过信号验证 → 样本外检验 → 环境分解 → 行为金融四层框架构建了完整的方法论体系——而不只是"写了一个回测"。
